Why it breaks Two common mistakes, and the fix
Mistake 1
The label gets cut off
Locking text to one line with white-space: nowrap means that at large sizes the words run past the edge and disappear. The button still looks tappable — but the label is gone, and no one notices in design review.
Mistake 2
The button can't grow
A fixed height: 44px can't stretch to fit bigger text. The label spills out of the button or gets clipped — and neighboring elements get shoved around.
The fix
Let it size to its content
Swap fixed pixels for min-height and em units, and let text wrap. The button grows with the type and the layout flexes around it — nothing clips, nothing hides.

What value this brings
Richer component documentation
Design and tech docs could show a component live across every accessibility view, not just one static frame — making the spec something you see, not just read.
Live prototype environments
Scale a real screen to any accessibility size on the fly — instantly see where content slips below the fold, where layouts expand, and where things break.
Beyond type, into vision
Imagine the same screen viewed through color-blindness simulations or quick translations — catching where contrast fails or longer copy reflows before it ships.
